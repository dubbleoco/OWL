01 — The Problem

ADHD EATS IDEAS FOR BREAKFAST

Running a one-person fabrication shop means your brain is always running. Customer orders, product ideas, machine settings to remember, things to research, tasks to schedule — it never stops. The problem isn't generating ideas. It's that the ideas don't wait for you to be at a desk.

They hit while you're mid-cut on the laser. While you're loading filament. While you're driving to a dump run. And with ADHD, if a thought isn't captured the moment it appears, it's gone — not filed away somewhere, actually gone.

  • 01 Ideas evaporate in seconds if there's no frictionless capture method within arm's reach
  • 02 Notes apps require context-switching — opening the app, choosing a category, typing correctly — which breaks flow
  • 03 Unorganized brain dumps (voice memos, random texts to yourself) pile up and never get acted on
  • 04 Tasks mixed with ideas mixed with reminders means nothing gets prioritized
  • 05 The cost of forgetting a good business idea or a critical machine setting is real money
The goal wasn't to build a better notes app. It was to build something that thinks alongside you — something you can just talk at and trust it to sort itself out.
02 — What OWL Is

YOUR BRAIN'S OUTBOX

OWL is an AI agent that lives inside a private Telegram bot. You send it anything — a half-formed idea, a task you just remembered, a quote you want to save, a reminder to call someone back — and it figures out what you meant and where it belongs.

No commands. No menus. No categories to select. Just fire off a message in plain English, voice-to-text, or a photo, and OWL handles the rest. It categorizes every input into a structured format called a pellet, tags it by type, and queues it for the appropriate downstream agent or system.

"Just talk at it.
It'll figure it out."

03 — The Pellet System

FIVE CATEGORIES.
ZERO DECISIONS.

OWL classifies every input into one of five pellet types. You don't tell it which type — it infers from context. A sentence like "maybe I should try resin casting" becomes an IDEA. "Call the supplier back" becomes a TODO. You never have to think about it.

The pellet feed

Every message you send OWL shows up in a structured feed — sorted by type, tagged by business line, timestamped. The feed is visible from the OpenClaw dashboard, but more importantly, it's processed automatically by the downstream agents.

IDEAs get queued for the relevant business agent's next review cycle. TODOs get assigned priority and scheduled. MEMORYs get written into agent context. REMINDERs get scheduled for ATLAS to surface at the right time. RESEARCH items get queued or immediately actioned.

The pellet feed is effectively the nervous system input for the entire OpenClaw stack.

Routing is determined by a combination of keyword context, the pellet type, and the sender pattern. OWL knows the shape of each business line in the OpenClaw stack — fabrication, photography, guiding, e-commerce, trading — and infers which agent should receive the pellet based on content alone.

For ambiguous pellets, ATLAS makes the final routing call. For unambiguous ones — like "add to tomorrow's BermSnap checklist" — OWL routes directly.

05 — Why I Built It

THE ADHD
ACCOMMODATION

The honest answer is that OWL was built because I was losing too much. Ideas that came during fabrication work. Machine settings I figured out through trial and error and then forgot. Tasks that surfaced at 11pm and evaporated by morning. The mental load of running six separate revenue streams with no administrative support is significant, and my brain doesn't file things neatly.

Every other solution I tried had too much friction. Opening a notes app requires a conscious decision to switch context. Voice memos pile up unprocessed. Sticky notes get buried. The thing that was missing wasn't storage — it was an intelligent receiver that never requires you to organize anything yourself.

OWL is not a productivity tool. It's an external working memory that speaks plain English and knows where everything belongs. The goal is: zero cognitive overhead at the moment of capture.

The OpenClaw system was designed from the start with ADHD accommodations in mind: structured scheduling, shame-free redirection, focus protection, minimal context-switching. OWL is the front door of that system. It's the reason you can be elbow-deep in a CNC job and still capture a business idea without losing either.

The other design principle baked into OWL is that it should never judge. No error messages, no "I don't understand," no requiring you to rephrase. If the message is ambiguous, OWL makes its best inference, shows its work in the response, and asks for a thumbs up if needed. The bar for capture is as low as possible on purpose.

Before OWL, the process looked like: think of something → decide if it's worth stopping for → if yes, context-switch to a note app → pick a category → type it out → close app → get back to work. Most ideas didn't make it past step two.

After OWL, the process looks like: think of something → send it to Telegram → keep working. That's it. The organizing happens asynchronously, without you. By the time you check in with ATLAS at the end of the day, your ideas are sorted, your tasks are queued, your reminders are scheduled, and your machine settings are saved.

It's not magic. It's just one good agent doing one job extremely well — which turns out to be the thing that makes all the other agents possible.

07 — Under the Hood

TECHNICAL
DETAILS

OWL runs as part of the OpenClaw multi-agent stack on a local M4 Pro Mac Mini. It's reachable via Telegram and operates as a persistent agent process managed by the OpenClaw runtime.

RUNTIME

OpenClaw agent runtime · Mac Mini M4 Pro (24GB) · Node-based agent process · Persistent session management via .openclaw/agents/owl/sessions/

CHANNEL

Telegram bot · Bound via openclaw agents bindings · Inbound messages from registered peer IDs only · Private bot, not public-facing

OUTPUT FORMAT

Structured JSONL pellets · Type + content + tags + routing metadata · Stored in session logs · Queryable by ATLAS for daily briefing

ROUTING

Content-based agent routing via ATLAS orchestrator · Direct routes for unambiguous targets · Fallback to ATLAS review queue for ambiguous pellets

OWL was the first agent deployed in the OpenClaw stack — intentionally. The principle behind OpenClaw is: get one agent producing real value before starting the next. OWL was chosen first because it's the agent that feeds all the others. Without a reliable thought capture layer, every other agent in the system is starting from an incomplete picture.

The agent's session logs at ~/.openclaw/agents/owl/sessions/ use the standard JSONL format with type: "message" entries. These logs are the source of truth for the OpenClaw observability dashboard and ATLAS's end-of-day summary generation.
